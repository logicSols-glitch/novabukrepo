## Step 0: What is Node.js?

Before we start, here's what's happening:
- **Node.js** = JavaScript that runs on your computer (not in a browser)
- **Backend** = The part that runs on your computer, stores data, and makes decisions
- **Frontend** = What users see (HTML, CSS, JavaScript in browser)
- **MongoDB** = A database (like a filing cabinet) that stores all your blog posts

Think of it like a restaurant:
- Frontend = The menu that customers see
- Backend = The kitchen where food is prepared
- Database = The storage freezer with ingredients

## Step 4: Configure Environment Variables

**This tells our backend WHERE to find the database and what secret code to use**

### 4.1 Locate the .env.example File

In your `backend` folder, you should see a file called `.env.example`

### 4.2 Copy It

1. Right-click on `.env.example`
2. Click **Copy**
3. Right-click in empty space in the same folder
4. Click **Paste**
5. You get a new file - rename it to `.env` (remove the `.example` part)

### 4.3 Edit the .env File

1. Right-click on your new `.env` file
2. Click **Open with** > **Notepad**
3. You should see something like:
   ```
   MONGODB_URI=mongodb://localhost:27017/novabuk-blog
   JWT_SECRET=your_super_secret_key_change_this
   PORT=5000
   NODE_ENV=development
   ```

### 4.4 Update MongoDB Connection

**If you used MongoDB Atlas (Cloud option):**

1. Replace the `MONGODB_URI=...` line with your connection string from Step 2
2. It should look like:
   ```
   MONGODB_URI=mongodb+srv://admin:admin123@cluster0.xxxxx.mongodb.net/novabuk-blog
   ```

**If you used Local MongoDB:**

Leave it as is:
```
MONGODB_URI=mongodb://localhost:27017/novabuk-blog
```

### 4.5 Save the File

1. Press **Ctrl** + **S**
2. Close the file

‚úì **Done! The backend now knows where the database is.**

## Troubleshooting

### Backend won't start / MongoDB Connection Error

**Problem:** You see `MongoDB Connection Error` or `ECONNREFUSED`

**Solution:**
1. **If using MongoDB Atlas:**
   - Check your connection string in `.env` is correct
   - Go to MongoDB Atlas website
   - Click "Clusters" ‚Üí "Connect" ‚Üí Check your IP is whitelisted
   - Verify username and password are correct

2. **If using Local MongoDB:**
   - Make sure MongoDB is installed
   - On Windows, MongoDB might run automatically, but check:
     - Press `Windows Key`, search for "Services"
     - Look for "MongoDB Server" and make sure it's "Running"

### Admin Dashboard Shows Blank Page

**Problem:** Page is white or empty

**Solution:**
1. Open browser **Developer Console:**
   - Right-click on page
   - Click "Inspect" or "Inspect Element"
   - Click "Console" tab
   - Look for red errors
2. Most likely issue: Backend isn't running
   - Go back to Command Prompt
   - Make sure you see the "üöÄ Backend running" message
   - If not, run `npm run dev` again

### Can't Find Admin Dashboard File

**Problem:** File:/// URL doesn't work

**Solution:**
1. Open File Explorer
2. Navigate to: `C:\Users\IDOKO NELSON EHI\Desktop\NOVABUK\first\main\`
3. Find `admin-dashboard.html`
4. Right-click it
5. Click "Open with" ‚Üí Choose your browser

### Blog Posts Don't Show Up

**Problem:** Blog page is empty or shows "No blog posts yet"

**Solution:**
1. Check you published the blog (not just saved as draft)
2. In Dashboard, look at the blog you created
3. Make sure the "Published" toggle is ON (green)
4. Refresh the blog page (press F5)

## What's Happening Behind the Scenes?

When you:

1. **Create a blog in admin dashboard:**
   - ‚úì Frontend sends your data to backend
   - ‚úì Backend checks you're logged in (authentication)
   - ‚úì Backend saves to MongoDB database
   - ‚úì You see "Success!" message

2. **Visit blog-dynamic.html:**
   - ‚úì Frontend asks backend "Give me all blogs"
   - ‚úì Backend checks MongoDB database
   - ‚úì Returns all published blogs as JSON
   - ‚úì Frontend displays them on the page

3. **Click on a blog to read:**
   - ‚úì Frontend asks backend "Give me this blog"
   - ‚úì Backend finds it in MongoDB
   - ‚úì Returns full content
   - ‚úì Frontend displays it
